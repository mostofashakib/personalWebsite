Activation Functions

To understand why these models train at all, you need one more neural network primitive: the activation function. After each layer computes a weighted sum of its inputs, the model applies an activation function to inject non-linearity. Without this step, a deep network would mathematically collapse into something equivalent to a single linear transformation, no matter how many layers you stack.

One of the simplest and most influential activation functions is ReLU:

ReLU(x) = max(0, x)

This means any negative value gets snapped to zero, while any positive value passes through unchanged. That simple thresholding behavior makes gradient-based training much easier than older saturating functions like sigmoid or tanh, which often squash values into narrow ranges and cause gradients to vanish.

ReLU matter in practice because it gives the network a clean way to build sparse, selective feature detectors. A neuron can remain completely inactive for one pattern and activate strongly for another. Across billions of parameters, this helps the model form useful internal circuits for syntax, semantics, and higher-level abstractions.

That said, modern transformers often use GELU instead of raw ReLU. GELU is smoother and tends to work better in large language models because it softly gates activations rather than applying a hard cutoff. But the core idea is the same: activation functions are what stop the network from being a giant linear calculator and allow it to model complex structure.
